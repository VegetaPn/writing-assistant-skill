# Case: Markdown 加粗后需加空格 + 公众号长文加粗密度不足

**Date:** 2026-02-25
**Skill/Step:** writing-assistant Step 8 (Final Article)

## Context
用户要求在公众号终稿中增加正文加粗，并指出 `**加粗文字**` 后面直接跟中文字符时渲染可能失败。

## AI Output
1. 全文约 3000 字，仅 9 处加粗，对公众号长文来说偏少
2. 多处加粗标记 `**` 后直接跟逗号、句号或中文字符，无空格：
   - `**机器身份（Machine Identity）**，`
   - `**这不是一个技术问题。这是一个制度问题。**技术`
   - `**生态耦合度（Ecosystem Coupling）**。`

## User Correction
1. "正文里面加粗的部分可以稍微多一点吗，微信长文加粗感觉挺重要的"
2. "这种加粗应该在第二个**后面衔接一个空格，否则有时候渲染不出来"

## Root Cause
- 风格偏差: 公众号长文的加粗密度认知不足，没有把每个章节的核心论点都加粗
- 错误方法: Markdown 加粗语法的渲染兼容性处理不当

## Lesson
1. 公众号长文每个章节至少有 1-2 处关键论点加粗，全文加粗密度约每 200 字一处，便于读者扫读。
2. Markdown 加粗标记 `**` 闭合后，如果后面还有字符（非换行），必须加一个空格再接后续内容，否则部分渲染器无法正确解析。
