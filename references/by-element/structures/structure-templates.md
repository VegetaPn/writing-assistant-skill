# Structure Templates

> 从优秀文章中提取的整体结构模式，分析其逻辑流、章节组织和叙事弧线。

## Structure #1: The future of work when work is meaningless

**Source:** Dan Koe (X/Twitter, 2026-01-21)

### 章节概览

| 章节 | 标题 | 核心功能 |
|------|------|---------|
| Opening（无标题） | — | 焦虑共鸣 + 问题轰炸 + 价值承诺 |
| I | The meaning crisis makes meaning worth paying for | 历史背景铺垫：意义是如何丢失的 |
| II | The acceleration of AI (and meaninglessness) | 现状诊断：AI如何加速问题 |
| III | The evolution and anatomy of meaning (robots vs humans) | 理论框架：意义的生成机制 |
| IV | The creator economy = the meaning economy | 机会识别：新经济形态 |
| V | The last defensible moat is you | 核心论点：不可替代性在哪里 |
| VI | The post-labor skill stack | 行动方案：具体该怎么做 |

**总计：6个正式章节 + 1个开头段落，约4800字**

### 逻辑递进模式（Progression Pattern）

```
背景/历史 → 现状/问题 → 理论/框架 → 机会/可能性 → 核心论点 → 行动指南
```

详细拆解：

1. **铺垫层（Section I-II）**：从人类文明史的角度解释"意义"是如何一步步丢失的，再引入AI如何进一步加速这个趋势。这两节的功能是让读者理解问题的全貌和深度。

2. **理论层（Section III）**：提出"意义的解剖学"——意义由什么构成（进步 + 贡献），由什么驱动（挣扎、好奇、地位）。这一节为后续的论证提供了理论基础。

3. **机会层（Section IV）**：将理论应用到现实中——创作者经济就是意义经济。这一节的功能是让读者看到出路。

4. **核心层（Section V）**：提出"Swap Test"概念——如果换一个人创作，作品是否同样有价值？这是全文的核心洞察。

5. **行动层（Section VI）**：将所有理论转化为可操作的技能层级（agency → taste → perspective → persuasion → technical know-how）。

### 章节命名惯例

- 使用**罗马数字编号**（I、II、III...）
- 标题风格：**概念性/描述性**，而非指令性
- 标题中常包含**对比或张力**（"robots vs humans"、"meaninglessness"）
- 有时使用**等式结构**（"The creator economy = the meaning economy"）

### 开头、发展、收尾方式

- **开头**：以时代焦虑为锚点，用连续提问制造信息缺口，最后承诺"6个想法"
- **发展**：从宏观历史逐步聚焦到个人行动，每一节比上一节更具体、更实用
- **收尾**：以简短的技能清单结束，附带一句鼓励实验的话。没有总结性段落，而是以"最佳建议"自然收尾，签名"– Dan"

### 结构特征总结

- **宏大叙事 → 个人行动**：从文明史讲到个人技能栈，跨度极大但逻辑连贯
- **每一节都有独立价值**：即使单独阅读某一节，也能获得有意义的洞察
- **引用和理论嵌入**：引用 David Shapiro、Chris Paik、Naval 等人的理论，增加权威性和知识密度
- **渐进式揭示**：核心答案（你的视角不可替代）不在开头给出，而是通过层层论证在第五节才揭示

## Structure #2: The most important skill to learn in the next 10 years

**Source:** Dan Koe (X/Twitter, 2026-01-08)

### 章节概览

| 章节 | 标题 | 核心功能 |
|------|------|---------|
| Opening（无标题） | — | 断言开场 + 揭示答案 + 价值承诺 |
| I | Agency is the ability to iterate without permission | 定义核心概念 + 三个子论点 |
| II | AI is not a threat to the high agency | 将概念应用到AI焦虑场景 |
| III | Why generalists win in the AI age | 延伸论证：通才 vs 专才 |
| IV | The 5 human capabilities | 理论支撑：人类不可替代的底层能力 |
| V | How to actually practice agency | 行动指南：如何练习 |

**总计：5个正式章节 + 1个开头段落，约3200字**

### 逻辑递进模式（Progression Pattern）

```
定义概念 → 解除恐惧 → 扩展视角 → 理论支撑 → 实践方法
```

详细拆解：

1. **定义层（Section I）**：用"不是什么"来定义"是什么"。先说 agency 不是什么（不是机械服从），再通过三个子论点（iterate without permission / life as experiment / believe in the difficult）正面定义。

2. **应用层（Section II）**：将 agency 概念直接对准读者最大的恐惧——AI威胁。论证高 agency 的人不会被AI取代。

3. **扩展层（Section III）**：从"个人agency"扩展到"通才vs专才"的讨论，论证为什么多元技能比单一专精更有未来。

4. **理论层（Section IV）**：列出人类的5种基本能力，论证AGI无法在根本上超越人类。

5. **行动层（Section V）**：给出具体的练习方法，以社交媒体作为"练习agency的游戏"来收尾。

### 章节命名惯例

- 同样使用**罗马数字编号**（I、II、III...）
- 标题风格更偏向**论点式**（直接用一句话表达该章节的核心观点）
- 示例："Agency is the ability to iterate without permission" — 标题本身就是一个可引用的观点
- 示例："AI is not a threat to the high agency" — 直接表明立场

### 开头、发展、收尾方式

- **开头**：大胆断言 + 自我修正 + 直接给出答案（agency）+ 承诺"5个想法"
- **发展**：从概念定义逐步扩展到更广的应用场景，但始终围绕"agency"这一核心
- **收尾**：以社交媒体作为"练习agency的最佳游戏"来收尾，将理论落地为具体的行动建议，签名"– Dan"

### 结构特征总结

- **单一核心概念展开**：整篇文章围绕一个概念（agency）展开，不像第一篇那样涉及多个主题
- **子论点分层**：Section I 内部有三个编号子论点（1、2、3），这是文章内部的微观结构
- **引用穿插**：每个章节开头或中间引用名人语录（Krishnamurti、Devon Eriksen、C. Thi Nguyen），既增加权威性又打破纯文字的单调
- **对比论证**：大量使用"高 agency vs 低 agency"的对比来帮助读者理解

## 两篇文章的结构对比

| 维度 | 文章一（Future of Work） | 文章二（Most Important Skill） |
|------|------------------------|-------------------------------|
| 章节数量 | 6节 | 5节 |
| 文章长度 | 更长（约4800字） | 较短（约3200字） |
| 核心答案揭示时机 | 延迟到第五节 | 开头就揭示 |
| 叙事跨度 | 文明史 → 个人行动 | 概念定义 → 实践方法 |
| 内部结构 | 每节平铺展开 | 部分章节有编号子论点 |
| 引用方式 | 引用理论和人物 | 引用名言金句 |
| 结尾方式 | 技能清单 + 实验建议 | 社交媒体实践 + 具体步骤 |

## Dan Koe 的通用结构模板

```
[开头：焦虑共鸣 + 独特立场 + 价值承诺]

Section I:   铺垫/定义（让读者理解问题或概念）
Section II:  深化/诊断（引入新的维度或威胁）
Section III: 理论/框架（提供理解的工具）
Section IV:  机会/应用（让读者看到出路）
Section V:   核心洞察/论点（最有力的观点）
Section VI:  行动指南（可操作的下一步）[可选]

[结尾：简短收束 + 签名 "– Dan"]
```

**关键原则：**

- 罗马数字编号创造正式感和"论文"质感
- 每一节逐步从宏观走向微观，从理论走向实践
- 最后一节总是面向行动（"how to"、"skill stack"、"practice"）
- 文章末尾不写正式总结，而是以一句简短的鼓励自然收束

## Structure #3: 层递式分解结构（dontbesilent）

**Source:** dontbesilent 聊赚钱
- 《赚钱的第一步，是解决掉自己逻辑混乱的问题》(小红书/抖音)
- 《去年涨粉 70 万后，我所有方法压缩成了一句话》(小红书/抖音)

### 章节概览

| 层级 | 功能 | 核心操作 |
|------|------|---------|
| 开头 | 反常识断言 + 方法预告 | 抛出核心结论，承诺拆解过程 |
| 第一层 | 问题框架：识别目的与手段 | 给出分析工具（"区分目的和手段"） |
| 第二层 | 案例展开：逐层深入检验 | 用 2-3 个具体案例演示工具如何使用 |
| 第三层 | 元认知升级：从思考到"原思考" | 从案例中提炼出更高阶的认知方法 |
| 收尾 | 回归本质 + 身份筛选 | 不总结，而是用一句话点明"什么样的人才能做到" |

### 逻辑递进模式（Progression Pattern）

```
反常识开场 → 给出分析工具 → 案例验证（逐层深入） → 元认知升级 → 身份筛选收尾
```

详细拆解：

1. **工具层（第一层）**：不直接讲道理，而是先给读者一个分析工具。如 Article 108 中的"区分目的和手段"，Article 017 中的"找一个足够大、足够有价值、没有被研究清楚的领域"。工具的功能是让读者获得一种可以反复使用的判断能力。

2. **验证层（第二层）**：用具体案例逐层展开，每个案例都从上一层的结论出发，层层递进。如 Article 108 中"涨粉≠赚钱 → 起号方法≠变现方法 → 搬运≠IP"，Article 017 中"足够大=选题多 → 足够有价值=有人付费 → 没研究清楚=语言不过关"。每一层都比上一层更具体，但结论反而更反常识。

3. **元认知层（第三层）**：从"教你一个方法"升级到"教你如何生产方法"。如 Article 108 中从"识别目的和手段"升级到"原思考——思考我是如何思考的"；Article 017 中从"写好标题"升级到"如果事情本身足够有价值，标题只需平铺直叙"。

4. **身份筛选层（收尾）**：不写总结段，而是用反向筛选收尾。如"如果你意识到了这一点，你也不需要来抄我了""如果这件事情平铺直叙出来没人看，那我就不发了"。

### 与 Dan Koe 线性递进的对比

| 维度 | Dan Koe 线性递进 | dontbesilent 层递式分解 |
|------|-----------------|----------------------|
| 推进方式 | 章节之间线性前进（背景→理论→行动） | 层层剥洋葱，每层从上一层结论出发 |
| 核心工具 | 提出概念（如 agency），逐步论证 | 提出分析方法，用案例反复验证 |
| 章节关系 | 每节相对独立，可单独阅读 | 每层依赖上一层，必须顺序阅读 |
| 读者参与 | 读者被说服（理论→认同） | 读者被训练（工具→使用→升级） |
| 元认知 | 较少，停留在"理解概念"层面 | 核心特征，要求读者"学会如何思考" |
| 收尾方式 | 行动清单 + 签名 | 身份筛选 + 反向激励 |
| 引用风格 | 引用名人/理论增加权威性 | 引用自己的爆款数据增加可信度 |
| 口语感 | 书面化，长句，学术词汇 | 高度口语化，短句，反问极多（"对吧？"） |
| 章节编号 | 罗马数字（I、II、III） | 无正式编号，用"第一层/第二层"或逻辑递进 |

### 结构特征总结

- **教方法而非教知识**：每篇文章的核心产出不是一个观点，而是一个可以反复使用的分析工具
- **反常识是燃料**：每一层的推进都依靠"你以为的常识其实是错的"来驱动
- **口语化外壳+严密逻辑内核**：表面上像朋友聊天，实际上是严格的逻辑推导链
- **自我数据背书**：用自己账号的播放量、粉丝数作为论据，而非引用名人
- **身份筛选收尾**：不鼓励所有人行动，而是用"如果你做不到这一点，你也不需要做"来反向筛选
