## Method #1: 底层逻辑提取法（提取逻辑而非内容）

**Core Framework（核心框架）：**

```
┌─────────────────────────────────────────────────────────┐
│              AI 内容创作正确路径                           │
│                                                         │
│  ❌ 错误路径: 内容 → AI → 内容（直接让AI仿写）            │
│  ✅ 正确路径: 内容 → AI提取底层逻辑 → 用逻辑产出新内容     │
│                                                         │
│  Step 1: 收集爆款内容（文稿/封面/标题）                    │
│  Step 2: AI 基于心理学/传播学提取底层逻辑                  │
│  Step 3: 用底层逻辑创作属于自己的内容                      │
│                                                         │
│  核心区别: 掌握"方法的方法"（原方法），而非方法本身          │
└─────────────────────────────────────────────────────────┘
```

[详细文档](extract-logic-not-content.md)

**Source（来源）：**
- dontbesilent -《我涨粉 40 万没花一分钱》(小红书/抖音, Article 085)
- dontbesilent -《做短视频的本质是黑掉人的认知系统》(小红书/抖音, Article 033)

**Psychology Mechanism（心理学机制）：**

1. **元认知能力**：AI 擅长从大量样本中提取模式，但不擅长直接模仿人类的创作风格。利用 AI 做模式提取而非内容生成，恰好匹配了 AI 的能力边界。
2. **迁移学习原理**：从具体案例中抽象出底层逻辑，再将逻辑应用到新场景——这是人类学习的核心机制，AI 加速了"抽象"这一步。

**Why It Works（为什么有效）：**

> "所有人以为 AI 用来写稿子......很多人会直接让 AI 写稿子，而这个是不会用 AI 的一个表现。"

- 直接让 AI 仿写 → AI 味重，且无法持续产出差异化内容
- 让 AI 提取底层逻辑 → 你获得了一个"方法工厂"，可以持续产出原创内容
- 底层逻辑一旦提取出来，用 AI 或用人执行最后一步都可以

**Practice Guide（实践指南）：**

1. 收集目标博主的爆款内容（文稿/封面/标题）
2. 分门别类整理后发给 AI
3. 提示词关键指令："基于心理触发机制、认知偏差、传播学、营销心理学、注意力经济，挖掘这些内容背后的底层逻辑"
4. AI 输出方法论框架后，用框架指导自己的创作
5. 不断验证和迭代框架

**Case Study（案例分析）：**

- **正面案例**：dontbesilent 用此方法从其他博主的爆款中提取出"四因子评分系统"，然后用这个系统指导自己的短视频开头，实现了 9 个月涨粉 40 万
- **反面案例**：直接让 AI 仿写某博主的文案 → "不管你用 GPT4 还是 GPT5 都不可能"

## Method #2: AI 导师系统（布鲁姆两标准差 + Claude Code 文件系统）

**Core Framework（核心框架）：**

```
┌─────────────────────────────────────────────────────────┐
│              AI 导师系统                                  │
│                                                         │
│  理论基础: 布鲁姆两标准差效应                               │
│  ─ 一对一导师指导 vs 传统课堂 → 领先 2σ（前 2%）          │
│                                                         │
│  实现条件:                                               │
│  1. 智慧的导师 ──── AI 模型（Claude / GPT）               │
│  2. 定制化推进 ──── Claude Code 文件系统记忆学习进度        │
│                                                         │
│  人机分工:                                               │
│  人 → 提出问题、判断方向、做不可替代的决策                  │
│  AI → 提取逻辑、整理信息、做可复制的执行                   │
└─────────────────────────────────────────────────────────┘
```

**Source（来源）：**
- dontbesilent -《真心建议普通人尽快让 AI 做人生导师》(小红书/抖音, Article 083)
- dontbesilent -《出版社让我出书，我说今年戒掉了读书》(小红书/抖音, Article 001)

**Psychology Mechanism（心理学机制）：**

布鲁姆（Benjamin Bloom）的研究证明：一对一导师指导的学生比传统课堂学生平均领先两个标准差（约前 2%）。关键条件是导师能根据学习者的具体情况定制化推进。AI + 文件系统的组合首次让普通人也能获得这种定制化的一对一指导。

**Why It Works（为什么有效）：**

> "不是大家所想象的那种说把一个 PDF 丢给一个 AI 知识库，然后你像压缩饼干一样速读一下，不是那么简单的概念。"

- 传统读书：线性、被动、无法根据理解程度调整
- AI 导师：交互式、主动提问、根据回答动态调整难度
- Claude Code 文件系统：记忆学习进度，不会每次从零开始

**Practice Guide（实践指南）：**

1. 确定学习领域
2. 让 AI 基于你的当前水平设计学习路径
3. 通过提问与 AI 交互学习，而非单向阅读
4. 利用文件系统保存学习进度和已掌握的知识
5. 定期让 AI 根据进度调整学习方向

**Case Study（案例分析）：**

- **正面案例**：dontbesilent 的工作目录有 320+ 文件夹、3000+ 文件、191 万字，全部由 AI 创建、编辑、整理、记录。"即便是最简单的创建文档这么一个事，我也觉得我不应该有权限去做"
- **反面案例**：把 PDF 丢给 AI 知识库速读 → 只是更快的阅读，没有利用 AI 的交互能力

## 如何添加新的方法论

1. **从文章/分享中识别方法论：**
   - 是否包含一个关于如何使用 AI 的核心框架？
   - 是否区分了 AI 的正确用法和错误用法？
   - 是否有通过此方法拿到结果的数据案例？

2. **提取并结构化：**
   - 按照上面的格式模板填写
   - 确保包含"为什么直接让 AI 做不行"的反面论证
   - 提供可操作的提示词模板或工作流

3. **关联到写作流程：**
   - 这个方法论应该在 Step 2（搜索参考）和 Step 6（Polish）阶段使用
   - 与整体写作流程的 AI 辅助环节配合
